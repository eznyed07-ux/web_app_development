# 產品需求文件 (PRD) - 賽特選房系統

## 1. 專案概述

### 背景與動機
在賽特等具備高隨機性與週期性的遊戲中，玩家往往需要耗費大量時間手動盯盤、選擇適合的房間進行遊戲，並面臨情緒化決策引起虧損的風險。「賽特選房系統」旨在提供自動化的選房、操作、停損及出金機制，並透過爆分預警功能，協助玩家能夠以更穩健的方式參與遊戲，並最大程度確保獲利能力，實現「穩定出金」的目標。

### 目標用戶
- **想穩定獲利的賽特玩家**：希望有系統輔助，不用依靠純運氣或直覺操作。
- **無暇長時間盯盤的使用者**：需要自動化腳本代為進行遊戲操作與出金管理。
- **投資心態的玩家**：需要嚴格執行停損停利策略的使用者。

### 核心價值主張
- **省時與自動化**：免去人工找房、操作的時間成本。
- **風險控管**：系統化自動執行停損，避免非理性的超額虧損。
- **提高勝率與獲利**：利用數據分析爆分時機預警，結合穩定出金策略落實收益。

---

## 2. 功能需求

### 主要功能與使用者故事

#### 1. 選房功能
- 作為 **玩家**，我希望 **系統能自動推薦或篩選出勝率高的房間**，以便 **我能在最佳的賠率週期進場**。

#### 2. 穩定出金功能
- 作為 **玩家**，我希望 **系統獲利達到一定條件時能自動執行出金或提醒出金**，以便 **確保我的資金入袋，落實穩定出金的目標**。

#### 3. 自動設定停損點功能
- 作為 **玩家**，我希望 **能夠設定每一局或每天的最高虧損金額上限**，以便 **系統在觸及停損點時自動停止下注，避免資金見底**。

#### 4. 自動操作功能
- 作為 **玩家**，我希望 **系統能夠代替我自動按鈕下注、依策略自動遊玩**，以便 **我可以離開電腦處理其他事情，仍能持續運作**。

#### 5. 爆分提前通知功能
- 作為 **玩家**，我希望 **系統在偵測到即將爆分 (Jackpot/大獎) 週期時能發送通知給我**，以便 **我可以關注進場或調整下注額度來提高獲利**。

## 3. 非功能需求

### 技術限制
- 本專案開發主要使用 **Flask + Jinja2 + SQLite** 作為網頁系統的基礎技術棧，提供玩家管理介面、報表查看與參數設定。
- 需搭配自動化操作技術來驅動實際遊戲流程（如透過爬蟲或 API 控制等方式）。

### 效能與安全考量
- **即時性（Latency）**：爆分通知與自動停損需要低延遲處理，以確保在極短的遊戲週期內做出反應。
- **安全性**：使用者的帳號密碼與憑證需加密儲存於資料庫，防止被盜取而造成真實資金損失。
- **穩定性**：自動操作遇到網路斷線或遊戲介面更動時，需要具備重新連線或安全退出機制的保護，不能無故持續失誤扣款。
- **隱私性**：確保出金與獲利紀錄不被第三方輕易截取。

## 4. MVP 範圍

### Must Have (必須具備 - 第一階段核心)
- 基礎的**選房功能**（提供簡易的選房介面或自動切換房間邏輯）。
- **自動設定停損點功能**（保護玩家本金不可或缺的功能）。
- 基本的**自動化操作**（可以啟動自動掛機遊玩）。

### Should Have (應該具備 - 第二階段優化)
- **快要爆分提前通知**（介面提示聲音或串接 Line 推播提醒）。
- **穩定出金功能**（當日獲利達標自動停止或導向出金流程）。

### Nice to Have (有會更好 - 未來展望)
- 自訂更進階的選房策略腳本與自動回測系統。
- 個人的歷史盈利儀表板與資金利用率分析報表。
